| Configuration | Description | Default | Should leave it at default? |
| --- | --- | --- | --- |
| `acks` | The number of partition’s replicas need to acknowledge the request before the producer receive and acknowledge response for `send`. | all | No, changing the value will impact the delivery guarantees. |
| `max.block.ms` | Timeout bounds the total time waiting for buffer allocation | 60000 | Yes, set this to Long.MAX\_VALUE for delivery guarantees |
| `buffer.memory` | The allocated buffer where records is waiting to be sent to the broker. | 33554432 | Maybe, check [Internal Link: our recommendations](producer_settings.html). |
| `retries` | Number of times to resend same batch when error detected. | 2147483647 | No, changing the value will impact the delivery guarantees. |
| `max.in.flight.requests.per.connection` | The maximum number of unacknowledged requests the client will send on a single connection before blocking | 5 | Yes, change this to 1 will impact the record order. |
| `batch.size` | The producer will attempt to batch records together into fewer requests whenever multiple records are being sent to the same partition. | 16384 | Maybe, increasing this for better throughput. |
| `linger.ms` | The time to wait before sending data to brokers. | 0 | Maybe, adjust this will impact latency |
| `compression.type` | The compression type for all data generated by the producer. | none | Maybe, enabling this may improve throughput. |

For more details about producer configuration check [External Link: Upstream Producer Configs](https://docs.confluent.io/platform/current/installation/configuration/producer-configs.html).
