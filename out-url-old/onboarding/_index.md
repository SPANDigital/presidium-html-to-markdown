This page documents all aspects of the on-boarding process. The
on-boarding process is designed to make sure we capture all key details
needed to properly support each use-case (i.e. supported clients,
cluster sizing, data modeling, etc.). In addition, we use the
opportunity to illuminate
[Concepts]({{< ref "concepts" >}}) you should be
comfortable with prior to using ACI Kafka.
