Roadmap¶

Note

Here you can find our current plan for customer facing features and developments. We periodically review and refine our roadmap according to customer needs and organizational priorities. This means it is subject to adjustments. If you have dependencies on anything in our roadmap or have any questions, please reach out to Tom Duckering.

Present¶

FY2020 - Quarter 4¶

  • Automated Onboarding for Playgrounds Providing fast, easy onboarding for small, non-production use cases via integration with SAFE.

  • Stateless Services in K8s Run our stateless, supporting services in K8s

  • Kafka Upgrade Upgrade Kafka to bring new features to improve reliability and performance. See Kafka 2.4.1 Release Notes

  • JDK 11 JARs Provide JDK 11 only JARs to complete our move away from JDK 8

  • Decommission Coda Close down vestigial use cases of Coda and move customers to supported alternatives, i.e Kafka.

  • Decommission Kafka Connect migrate all use cases onto the new Kafka Mirrors solution.

Future¶

FY2021 - H1¶

Note: This list is partial and to be confirmed

  • Golden Gate following on from Silk Road, we will offer ACI Kafka in Golden Gate.

  • Kafka Mirrors for Golden Gate the facility to replicate topics from Golden Gate based clusters.

  • Kube an offering of our service running on Kube for faster onboarding and elastic capacity.

  • API Improvements we plan to make some major improvements to our API to improve it’s reliability and usability.

If there is anything you would like to see us do, please contact us

Past¶

Below are the developments delivered in prior years.

2020¶

  • SilkRoad Build Out Delivering ACI Kafka for use in Silk Road, in preparation for Golden Gate.

  • Mirrors for SR Allow replication of topics from SilkRoad to AODCs - limited availability

  • SLOs Integrate with slo.apple.com for Kafka to provide visibility of our platform reliability

  • Improved Data Rebalancing added the ability to rebalance data between brokers to improve disk utilization

2019¶

  • JDK11 Compatibility

  • Public API available to all for self-service entity management

  • Integration with Topic Key Management Service Front End service (Iris) to eliminate C* client dependency

  • APIs and plan to facilitate the deprecation of the PIE Envelope in favor of Kafka Message Headers

  • Utilization reports for storage, CPU, and other Kafka and system resources

  • Kubernetes support to manage Kafka cluster running on K8s for SR and GG

2018¶

  • Production cluster available for Phase 1 and 2

  • Entity management is fully done with Portal and is backed by PIE Workflow.

  • Users can use portal for most actions necessary

  • Published SLOs about service reliability and reviewing them monthly.

  • Kafka ACLs enforcement with support for prefixed principle and resource identifiers

  • Configuration injection with Kaffe to enable easier setup for user

  • Kafka for All to allow polyglot use of ACI Kafka

  • Upgrade to Kafka 1.1

  • Inspector release to allow customers to troubleshoot problems and see their consumer lag.

  • Quota accounting in place which will enable us to enforce it in the future.

  • Limited access preview to mirroring solution based on Kafka connect.

2017¶

  • First Shared QA cluster available for Phase 1 customers in June

  • Begin placing production hardware orders for dedicated high-volume clusters

  • Early Availability for the ACI Kafka client

  • New Shared QA clusters added for Phase 2 customers

  • End-to-end encryption available to customers

  • First Shared Production cluster available for Phase 1 customers

  • First dedicated production clusters live
