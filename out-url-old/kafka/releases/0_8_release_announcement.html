ACI Kafka 0.8 Release Announcement¶

The ACI Kafka team is excited to announce the release of ACI Kafka 0.8. This release contains a number of new features, fixes, and improvements in the ACI Kafka client, including a drop-in compatible Kafka API, group.id convention enforcement, and secure delivery of client configuration secrets. We’ve also added a new Kafka Sidecar that can be used in development environments for teams needing a way to integrate locally with Kafka.

We’re also thrilled to announce the soft launch of Kafka Portal – a new and easy way to manage your ACI Kafka Topics and Clients. We’ll be reaching out to teams with log-in instructions as Kafka Portal availability ramps up over the next few weeks.

For the latest documentation, including an overview of ACI Kafka and the ACI Kafka Client, please see visit our documentation.

Notable changes¶

Client Update¶

In 0.8 the ACI Kafka Client was updated to better integrate into other services and frameworks (e.g. MirrorMaker, Spark, etc.). You can read more about these changes in the New ACI Kafka Client QTIP.

Summary of changes –

  • PieKafkaConsumer and PieKafkaProducer are deprecated in 0.8 and will be removed in 0.9

  • ConfigurationInterceptor is a new interface to allow for injecting configuration via Kaffe or other custom cluster discovery mechanism. See our docs for more information on ConfigurationInterceptors.

  • ACI Kafka Client now shades internal dependencies, and we now recommend you remove transitive inclusion of Apache Kafka from your dependencies. See our instructions for excluding Apache Kafka from your transitive dependencies.

Kafka ACLs & Consumer group id convention¶

In 0.8 we will begin enforcing server-side ACLs, and the client now requires consumer group names (i.e. group.id) to conform to a new naming convention. We’ve begun reaching out to teams to coordinate migrating offsets for changing consumer groups.

In our next release (0.9), brokers will begin enforcing consumer group ACLs, so please plan to upgrade clients to 0.8 over the next few weeks.

Server-side ACL enforcement should be invisible to teams using ACI Kafka, but if you experience any issues or if you have any questions on the migration, please contact us.

Secure delivery of Kafka secrets¶

The ACI Kafka client now automatically receives and injects secrets for your SASL and end-to-end encryption configuration. Teams already using ACI Kafka were asked to provide key pairs for their clients, and now the private key file can be directly configured to allow Kaffe to automatically inject secrets.

See our docs for how to configure your client with private key files, and details on how to implement your own private key provider.

Kafka Sidecar¶

Local integration and testing with ACI Kafka is now possible with new Sidecar support. Sidecars can be used to bring-up Kafka and Zookeeper locally, and in future releases, Kafka Sidecar will include tools for local topic and client management. Later this spring, we’ll be adding support for ACI Kafka features like Kaffe cluster discovery, configuration injection, and end-to-end data security.

Get started with Kafka Sidecar, and discover other Sidecars in Rio’s Sidecar community repo.

Kafka Portal¶

Kafka Portal is a new interface for managing your ACI Kafka Resources. Portal is powered by the new Kafka Public API which will be made available to teams in a future release.

When teams on-board to ACI Kafka, the Queue team will create Customer Groups and Namespaces with authorization set such that your team and delegate teams can manage and/or view topics, clients, and access control.

You can read more about Customer Groups, Namespaces, Group Access Control, etc. in the Concepts overview. We’ll be reaching out to teams over the next few weeks to share links and further instructions for accessing Kafka Portal.

Conclusion¶

For more information on the ACI Kafka service offering, including features, caveats, and project road map, visit our overview section.

You can reach out to the ACI Kafka team with questions on this release or general inquiries related to ACI Kafka.
