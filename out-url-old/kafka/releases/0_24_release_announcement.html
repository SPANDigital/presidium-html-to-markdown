ACI Kafka 0.24 release announcement¶

The ACI Kafka team is excited to announce the release of ACI Kafka 0.24. This release is based on our internal build of Apache Kafka 3.2 and contains a number of new features, fixes, and improvements.

For the latest documentation, including an overview of ACI Kafka and the ACI Kafka Client, please see our documentation.

Upgrade Instructions¶

The ACI Kafka 0.24 client is available on Artifactory. If you are using an older version of the ACI Kafka client, please upgrade to the latest version.

Although this release brings no changes in the envelope deprecation process, we would like to remind you to migrate your application away from the envelope if you haven’t done so yet. Support for consuming messages in the envelope format is still available in this version. This continues to guarantee interoperability with older clients producing messages in the envelope format. However, as with the previous release, producing messages in this old format is no longer possible.

Please make sure you have fully migrated away from using PieEnvelope and the envelope wire format. For more details, please refer to the documentation on the PieEnvelope deprecation.

This release includes a number of improvements:

  • KIP-704 should allow for smoother leadership failovers by having the controller indicate to new partition leaders whether they’re being elected uncleanly.

  • KAFKA-7572 Producer should not send requests with negative partition id

  • KAFKA-9279 Silent data loss in Kafka producer

  • KAFKA-13418 Brokers disconnect intermittently with TLS1.3 - We have backported this fix to our internal builds of 3.0 and 3.1

  • KAFKA-13310 KafkaConsumer cannot jump out of the poll method, and the consumer is blocked in the ConsumerCoordinator method maybeAutoCommitOffsetsSync(Timer timer). CPU and traffic on broker side increases sharply

  • KAFKA-12841 NPE from the provided metadata in client callback in case of ApiException

  • KAFKA-12256 auto commit causes delays due to retryable UNKNOWN_TOPIC_OR_PARTITION

  • KAFKA-13435 Static membership protocol should let the leader skip assignment (KIP-814)

  • KAFKA-13782 Producer may fail to add the correct partition to transaction

  • KAFKA-13794 Producer batch lost silently in TransactionManager

For a full list of changes please consult the release notes for Kafka 3.2. Or, for a general overview on what’s new in Kafka 3.2 you might be interested in this post in the Apache Kafka blog.

For more information on the ACI Kafka service offering, including features, caveats, and project road map, visit our overview section.

You can reach out to the ACI Kafka team with questions on this release or general inquiries related to ACI Kafka.
