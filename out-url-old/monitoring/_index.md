This section of the documentation shows how to report stats to hubble
and (in the future) how monitoring works. It covers both the client side
and broker side configuration. If you are simply a user of ACI Kafka you
can ignore the parts related to brokers.

Some of the components we built are:

- kafka-hubble-reporter: A MetricsReporter for KafkaMetric to report
  them to hubble. It is used for clients and brokers to publish stats.
- kafka-server-hubble-reporter: A Wrapper around kafka-hubble-reporter
  to also publish Yammer stats generated by the brokers.
- kafka-client-hubble-dashboards: A set of templates to use for
  monitoring producer and consumer KPIs in Hubble
- broker metrics: Client activities and actual usage from the broker side.
  These metrics include consumer lag, bytes-in/out, request received by the broker per topic, etc.
